# Conventions

## File Naming

- Use `kebab-case` for all file and directory names
- Markdown files use `.md` extension
- ADR files follow the pattern `ADR-NNN-title.md`

## Document Structure

Every document should have:

1. A top-level heading matching the file name
2. A brief introduction paragraph
3. Clearly defined sections with headings
4. Links to related documents where applicable

## Markdown Style

- Use ATX-style headings (`#`, `##`, `###`)
- Use fenced code blocks with language identifiers
- Use tables for structured data
- Use ordered lists for sequential steps
- Use unordered lists for non-sequential items
